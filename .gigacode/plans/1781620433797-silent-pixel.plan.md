# План изменения отображения рецептов

## Проблема
1. Блок "Why these numbers" и ингредиенты не сворачиваются/разворачиваются
2. Количество порций предопределено, а не рассчитывается автоматически
3. Ингредиенты были перенесены внутрь аккордеона "Why these numbers" и стали не видны

## Решение

### 1. Сделать блок "Why these numbers" аккордеоном
**Файл**: `app/src/main/java/com/abdulin/nutritionapp/presentation/recipe/RecipeDetailScreen.kt`

- Обернуть `NutritionBreakdownSection` в аккордеон с кнопкой раскрытия/сворачивания
- По умолчанию блок развёрнут (detailsExpanded = true)

### 2. Сделать блоки ингредиентов аккордеонами
**Файл**: `app/src/main/java/com/abdulin/nutritionapp/presentation/recipe/RecipeDetailScreen.kt`

- Обновить `IngredientNutritionSection` для каждого типа ингредиентов
- Кнопка раскрытия показывает детальную калорийность ингредиентов
- По умолчанию блоки свёрнуты (expanded = false)

### 3. Автоматический расчёт порций
**Файл**: `app/src/main/java/com/abdulin/nutritionapp/presentation/recipe/RecipeDetailViewModel.kt`

- Сделать методы `calculateSuggestedServings()` и `calculateServingsRange()` публичными
- Эти методы уже существовали, но были приватными:
  - < 300 ккал → 1 порция
  - 300-600 ккал → 2 порции  
  - 600-900 ккал → 3 порции
  - > 900 ккал → 4 порции

### 4. Обновить UI для отображения автоматических порций
**Файл**: `app/src/main/java/com/abdulin/nutritionapp/presentation/recipe/RecipeDetailScreen.kt`

- Изменить отображение `servingsCount` в `NutritionBreakdownSection`
- Использовать `calculateSuggestedServings()` для отображения расчётного значения
- Добавить текст "~X servings (calculated from calories)" для пояснения

### 5. Вернуть ингредиенты в видимое место
**Файл**: `app/src/main/java/com/abdulin/nutritionapp/presentation/recipe/RecipeDetailScreen.kt`

- Добавить отдельный блок для ингредиентов (`recipe_ingredients_title`)
- Ингредиенты теперь отображаются после `NutritionBreakdownSection`
- Каждый ингредиент показывается с калорийностью
- Если ингредиентов нет - показывается сообщение "Ingredients are not available yet"

## Исправленные ошибки

### Ошибка: NumberFormatException: empty String
**Описание**: При инициализации `selectedServings` использовалась `mutableStateOf(formatPortionMultiplier(...))`, что приводило к пустой строке в некоторых случаях.

**Решение**: Изменён тип `selectedServings` с `MutableState<String>` на `MutableState<Double>`, что гарантирует правильное хранение числового значения.

**Изменения**:
- Использование `selectedServings` напрямую как `Double`
- `formatPortionMultiplier()` вызывается только для отображения в UI
- Упрощены операции с кнопками +/- (прямая работа с `Double`)

### Ошибка: Дублирование строки в strings.xml
**Описание**: Строка `recipe_ingredients_title` была определена дважды, что вызывало ошибку компиляции.

**Решение**: Удалена дублирующаяся строка.
