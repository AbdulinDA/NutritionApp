# План: Улучшение UI рецептов с автоматическим расчётом порций и сворачиваемыми секциями

## Общее описание
Улучшить UX отображения рецептов за счёт:
1. **Автоматического расчёта порций** на основе калорий и веса блюда
2. **Сворачиваемых секций** для менее важной информации (разбор питания)
3. **Умного выбора порций** с ограничением вариантов при выборе 1 порции

## Анализ текущей реализации

### Проблемы текущей реализации

#### 1. Ручной выбор порций в `NutritionBreakdownSection`
**Файл**: `RecipeDetailScreen.kt` (строки ~550-650)

Текущий код:
```kotlin
FlowRow(horizontalArrangement = Arrangement.spacedBy(8.dp)) {
    for (servings in 1..maxPreviewServings) {
        FilterChip(
            selected = selectedServings == servings,
            onClick = { selectedServings = servings },
            label = { Text(...) }
        )
    }
}
```

**Проблема**: Показывает все варианты от 1 до max (обычно 3-6), даже если в рецепте всего 1 порция.

#### 2. Нет автоматического расчёта порций
**Файл**: `RecipeService.java` (строка ~267)

```java
int servingsCount = mainRecipe.getServingsCount() != null && mainRecipe.getServingsCount() > 0
        ? mainRecipe.getServingsCount()
        : 1;
```

**Проблема**: API возвращает `servingsCount` как есть, без расчёта на основе калорий/веса.

#### 3. Отсутствие сворачиваемых секций
**Файл**: `RecipeDetailScreen.kt`

Нет компонентов с `LazyColumn`, `Modifier.expanded()` или аналогичными функциями для сворачивания.

## Решения

### Решение 1: Автоматический расчёт порций

#### Подход A: Расчёт на основе калорий (рекомендуется)

**Логика**:
- Если рецепта < 300 ккал → 1 порция
- Если 300-600 ккал → 1-2 порции  
- Если 600-900 ккал → 2-3 порции
- Если > 900 ккал → 3-4 порции

**Реализация**:

**Файл**: `RecipeDetailViewModel.kt`

```kotlin
data class RecipeDetailUiState(
    // ... существующие поля
    val suggestedServings: Int? = null,  // Автоматически рассчитанные порции
    val servingsRange: IntRange = 1..3   // Допустимый диапазон для выбора
)
```

**Файл**: `RecipeDetailScreen.kt`

```kotlin
// Вычисление автоматических порций
val autoServings = calculateSuggestedServings(recipe.totalCalories)
val servingsRange = calculateServingsRange(recipe.totalCalories)

@Composable
private fun calculateSuggestedServings(totalCalories: Double): Int {
    return when {
        totalCalories < 300 -> 1
        totalCalories < 600 -> 2
        totalCalories < 900 -> 3
        else -> 4
    }
}

@Composable
private fun calculateServingsRange(totalCalories: Double): IntRange {
    return when {
        totalCalories < 300 -> 1..1  // Только 1 порция
        totalCalories < 600 -> 1..2
        totalCalories < 900 -> 1..3
        else -> 1..4
    }
}
```

**Файл**: `strings.xml` (новые строки)

```xml
<string name="recipe_servings_auto_calculated">~%1$d порции (на основе калорий)</string>
<string name="recipe_servings_calculated_from">Рассчитано: %1$d ккал / ~%2$d ккал на порцию</string>
<string name="recipe_servings_per_serving">~%1$d ккал на порцию</string>
```

### Решение 2: Умный выбор порций (ограничение вариантов)

**Файл**: `RecipeDetailScreen.kt`

**Текущий код**:
```kotlin
val maxPreviewServings = maxOf(servingsBase, 3).coerceAtMost(6)
FlowRow(horizontalArrangement = Arrangement.spacedBy(8.dp)) {
    for (servings in 1..maxPreviewServings) {
        // ...
    }
}
```

**Новый код**:
```kotlin
// Использовать servingsRange из ViewModel
val maxPreviewServings = servingsRange.last
FlowRow(horizontalArrangement = Arrangement.spacedBy(8.dp)) {
    for (servings in servingsRange) {
        FilterChip(
            selected = selectedServings == servings,
            onClick = { selectedServings = servings },
            label = {
                Text(
                    stringResource(
                        if (servings == 1) R.string.recipe_serving_single else R.string.recipe_serving_multiple,
                        servings
                    )
                )
            }
        )
    }
}

// Отображение автоматического расчёта
if (autoServings != null && autoServings != selectedServings) {
    Text(
        text = stringResource(
            R.string.recipe_servings_calculated_from,
            (perServingCalories * selectedServings).toInt(),
            perServingCalories.toInt()
        ),
        style = MaterialTheme.typography.bodySmall,
        color = MaterialTheme.colorScheme.onSurfaceVariant
    )
}
```

### Решение 3: Сворачиваемая секция "Почему эти цифры"

**Файл**: `RecipeDetailScreen.kt`

**Функция `NutritionBreakdownSection` → `NutritionBreakdownSection` с expandable**

```kotlin
@Composable
private fun NutritionBreakdownSection(
    recipe: RecipeModel,
    composition: RecipeCompositionModel?,
    // Добавить параметр expanded
    expanded: Boolean = false,
    onToggleExpanded: () -> Unit
) {
    // В начале: заголовок с кнопкой раскрытия
    Row(
        modifier = Modifier.fillMaxWidth(),
        horizontalArrangement = Arrangement.SpaceBetween,
        verticalAlignment = Alignment.CenterVertically
    ) {
        Text(
            text = stringResource(R.string.recipe_nutrition_breakdown_title),
            style = MaterialTheme.typography.titleLarge,
            fontWeight = FontWeight.Bold
        )
        // Кнопка раскрытия
        IconButton(onClick = onToggleExpanded) {
            Icon(
                imageVector = if (expanded) Icons.Default.ExpandLess else Icons.Default.ExpandMore,
                contentDescription = if (expanded) "Скрыть" else "Показать"
            )
        }
    }
    
    Spacer(modifier = Modifier.height(8.dp))
    
    // Условное отображение контента
    if (expanded) {
        Column {
            Text(
                text = stringResource(R.string.recipe_nutrition_breakdown_subtitle),
                style = MaterialTheme.typography.bodyMedium,
                color = MaterialTheme.colorScheme.onSurfaceVariant
            )
            // ... остальной контент
        }
    }
}
```

**Аналогично для других секций**:
- `NutritionCompositionSection` (сворачиваемая)
- `IngredientNutritionSection` (сворачиваемая)

## Критерии успеха

1. **Автоматический расчёт**:
   - При открытии рецепта автоматически рассчитываются порции
   - Отображается объяснение: "Рассчитано: 1200 ккал / ~400 ккал на порцию"

2. **Умный выбор порций**:
   - При 1 порции в рецепте показывается только 1 вариант
   - При 2-3 порциях показывается диапазон 1..2 или 1..3
   - При >3 порциях показывается диапазон 1..4

3. **Сворачиваемые секции**:
   - Раздел "Почему эти цифры" сворачивается по умолчанию
   - Кнопка раскрытия/сворачивания работает корректно
   - Контент плавно появляется/исчезает

4. **UX улучшения**:
   - Пользователь не видит ненужные варианты порций
   - Расчёт порций понятен и объяснён
   - Интерфейс менее перегружен информацией

## Проверка

### Ручное тестирование:

1. **Тест расчёта порций**:
   - Открыть рецепт с <300 ккал → должно быть 1 порция
   - Открыть рецепт с 400-500 ккал → должно быть 1-2 порции
   - Открыть рецепт с 700-800 ккал → должно быть 1-3 порции
   - Открыть рецепт с >1000 ккал → должно быть 1-4 порции

2. **Тест умного выбора**:
   - Открыть рецепт с 1 порцией → должен быть только 1 вариант
   - Открыть рецепт с 2 порциями → должны быть 1 и 2
   - Открыть рецепт с 3 порциями → должны быть 1, 2, 3

3. **Тест сворачивания**:
   - Открыть рецепт → секция "Почему эти цифры" должна быть свёрнута
   - Нажать "Показать" → контент должен появиться
   - Нажать "Скрыть" → контент должен исчезнуть
