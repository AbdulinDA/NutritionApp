# Текст Выступления

Тема моей выпускной квалификационной работы: `разработка системы персонализированного планирования рациона питания с использованием машинного обучения и рекомендацией рецептов на основе выбранных продуктов`.

Разработанное приложение решает задачу не просто учета питания, а именно персонализированной поддержки пользователя. Система объединяет несколько взаимосвязанных функций: ведение дневника питания, учет продуктов, которые есть у пользователя дома, персональные рекомендации рецептов и генерацию адаптивного плана питания.

Сначала пользователь заполняет профиль: указывает свои базовые параметры, цель, например снижение веса или поддержание формы, а также пищевые предпочтения, ограничения и нелюбимые продукты. Это важно, потому что именно эти данные становятся основой персонализации системы.

Далее пользователь может работать с каталогом рецептов. В отличие от обычного поиска, в приложении реализованы персональные рекомендации рецептов. Они формируются не только по категории блюда, но и с учетом цели пользователя, его предпочтений, продуктов, которые уже есть дома, а также накопленных поведенческих данных. Таким образом, система предлагает не случайные рецепты, а более подходящие именно этому пользователю варианты.

Отдельно реализован сценарий рекомендаций на основе выбранных продуктов. Пользователь может открыть раздел холодильника и отметить продукты, которые у него есть. После этого система предлагает рецепты, которые можно приготовить из этих продуктов. Это особенно важно с практической точки зрения, так как позволяет уменьшать количество лишних покупок и упрощает принятие решений.

Ключевой частью работы является план питания. Пользователь может задать количество дней, дату начала и использовать выбранные продукты для персонализации. После этого система формирует план питания автоматически. При построении плана используются не только правила по калорийности и макронутриентам, но и модель машинного обучения, которая участвует в выборе наиболее подходящих рецептов.

При этом план не является жестким. В приложении реализован адаптивный подход: пользователь может заменить блюдо, удалить его, закрепить понравившийся вариант, скрыть рецепт из будущих рекомендаций или вручную добавить рецепт в конкретный день и слот. Также система умеет достраивать только пустые позиции плана, не затрагивая уже выбранные блюда. Это делает взаимодействие более удобным и ближе к реальным потребностям пользователя.

Еще одна важная часть системы — дневник питания. Пользователь может фиксировать фактически съеденные блюда, и эта информация используется в дальнейшем для адаптации поведения приложения. То есть система учитывает не только заранее построенный план, но и реальные действия пользователя.

Таким образом, практическая ценность разработки заключается в том, что приложение помогает пользователю:

- учитывать рацион питания;
- получать персональные рекомендации рецептов;
- использовать продукты, которые уже есть дома;
- автоматически формировать и гибко корректировать план питания.

Итог работы состоит в создании единой мобильной системы, которая соответствует заявленной теме ВКР, так как реализует персонализированное планирование рациона, использует машинное обучение в рекомендационном контуре и поддерживает подбор рецептов на основе выбранных продуктов.
